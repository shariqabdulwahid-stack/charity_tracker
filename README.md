# Charity Tracker

A secure CRUD web application built with **Node.js, Express.js, MongoDB/Mongoose, and EJS templating**.  
This project was developed for **INFR3120 Assignment 3** to demonstrate full Create‑Read‑Update‑Delete functionality with authentication.



## Project Overview
Charity Tracker is an online directory of charitable campaigns.  
- Visitors can browse campaigns, view fundraising progress, and contribute donations.  
- Logged‑in users (admins) can create, edit, and delete campaigns or donations.  
- The app demonstrates secure authentication, session handling, and polished UI with Bootstrap.



## Features
- **Home Page**: Splash page with call‑to‑action buttons.
- **Campaign Directory**: List of all campaigns with progress bars.
- **Campaign Detail Page**: Shows description, goal, raised amount, and donation history.
- **Donation Form**: Add new donations to a campaign.
- **Authentication**: Login, register, and logout with session persistence.
- **CRUD Operations**:
  - Create campaigns and donations
  - Read campaign/donation details
  - Update campaign info and donations
  - Delete campaigns/donations with confirmation modals



## Technology Stack
- **Backend**: Node.js + Express.js
- **Database**: MongoDB Atlas with Mongoose
- **Frontend**: EJS templating, Bootstrap 5, custom CSS
- **Authentication**: express‑session with MongoDB store
- **Deployment**: Render (cloud hosting)
- **Version Control**: GitHub public repository with `.env` ignored
